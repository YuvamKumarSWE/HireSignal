  ---
  3 Services

  gemini.service.js
  - Generates 5 interview questions tailored to role/level/job description
  - Evaluates the full transcript at the end → returns hire_probability (0–1),
  strengths, weaknesses, verdict
  - Has mock fallback if API is down

  elevenlabs.service.js
  - TTS: converts question text → MP3 audio using one of 6 preset voices
  (George, Adam, Sarah, etc.)
  - STT: converts user's recorded WebM audio → transcribed text

  interview.service.js
  - Core session logic: create, track, update, complete sessions
  - Randomly assigns a voice per session
  - Builds the full transcript for evaluation

  ---
  Auth Flow

  1. User hits /auth → clicks "Continue with Google"
  2. Firebase popup handles Google OAuth
  3. On success → redirected to /dashboard
  4. Note: Firebase user ID is not actually used for sessions — sessions use a
  temp "user_" + Date.now() ID. Auth is essentially a gate, not deeply
  integrated.

  ---
  Interview Flow (End-to-End)

  Dashboard → POST /start → Gemini generates 5 questions → session created
       ↓
  Interview page → GET /question → show question
       ↓
  User answers (text or voice) → POST /answer or /answer/audio
       ↓
  Loop until 5 questions done
       ↓
  POST /end → Gemini evaluates transcript → hire probability + feedback
       ↓
  Results page

  ---
  Key Things to Know as You Take This Over

  1. No persistent storage — if you want to save interview history, you'll need
  to add a DB
  2. Firebase is just a login gate — your new Firebase project just needs Google
   Auth enabled, nothing else
  3. All AI logic is in the 3 services — that's where the real work happens
  4. Sessions die on restart — normal for dev, but worth noting

  ---
  Project Summary — HireSignal

  One-liner

  ▎ Full-stack AI-powered interview simulation platform that conducts realistic 
  voice and text interviews using LLM-generated questions and returns an 
  AI-scored evaluation with hire probability.

  ---
  What It Does (Explain to Anyone)

  HireSignal lets users practice job interviews with an AI. You pick a role,
  level, and paste a job description. The system generates 5 tailored interview
  questions using Google Gemini. You answer in text or by speaking. At the end,
  Gemini analyzes your full transcript and scores you: hire probability
  percentage, strengths, weaknesses, and a verdict. Questions are read aloud by
  one of 6 realistic ElevenLabs voices. Voice answers are transcribed
  automatically.

  ---
  Technical Depth (For Interviews)

  Architecture:
  - Decoupled frontend/backend — React SPA communicates with an Express REST API
   over HTTP + WebSocket
  - Service-layer pattern: business logic lives in 3 isolated services
  (interview, gemini, elevenlabs) — controllers only orchestrate
  - Session-based state: each interview is a unique session tracked by ID,

  How to Talk About It in an Interview

  "Walk me through the architecture"
  ▎ "It's a React frontend hitting an Express REST API. The interesting part is 
  the AI pipeline — when a user starts an interview, we call Gemini with their 
  role and job description to generate 5 tailored questions. During the 
  interview, ElevenLabs handles both text-to-speech for reading questions and 
  speech-to-text for transcribing voice answers. At the end, we send the full 
  Q&A transcript back to Gemini for evaluation. I also added a WebSocket layer 
  with Socket.io so the server can push real-time events to the client — things 
  like transcription status and a server-authoritative interview timer."

  "What was the hardest technical challenge?"
  ▎ "The voice pipeline was tricky. The browser records audio as WebM via the 
  MediaRecorder API, which we send as multipart form data to the backend. The 
  backend passes that buffer directly to ElevenLabs STT. The challenge was 
  keeping the UI responsive during the 2–4 second transcription wait — that's 
  actually why I added the WebSocket layer, so the server can push a 
  'transcribing...' status event the moment the audio arrives, before the STT 
  call even returns."

  "Why Gemini over GPT-4?"
  ▎ "For this use case, Gemini 2.5 Flash hits the right balance of speed and 
  quality. The structured JSON output mode (responseMimeType: 
  "application/json") is also very clean — no parsing hacks needed. It was also 
  free to prototype with."
